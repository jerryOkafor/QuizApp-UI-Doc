A) Overview - top

Requirements

  1. Android Studio 3.2 Canary 15 or greater
  2. ANdroid 5.0 or greater

This theme is basicallay a set of xml layout files and the accompanying navigation stacks built ontop of a three activities using the new Navigation Architecure component. You can read more about it here This theme is built ontop of many libraries like the one show below:

  1. Dagger
  2. Android ViewModel
  3. Kotlin

The code is entirely written in Kotlin, so welcome to Kotlin. The theme is made up of 11 screens which include:

  1. ChatsFragment.kt
  2. EarnCoin.kt
  3. GameOver.kt
  4. Home.kt
  5. LeaderBoard.kt
  6. Login.kt
  7. Notifications.kt
  8. Profile.kt
  9. Quz.kt
  10. SettingsActivity.kt
  11. Splash.kt

All other data are basically mocked to suit the UI needs, you can change the data structure to suit you need and functionalities. More screens will come in subsequent updates.

B) Build The Project - top

Import Project

If you are here, it means that you have unziped the project, so it is time to build the project and see what the theme is like. Follow thw outlined staps shown below and import the project into Android Studio:

  1. Click File -> New Project -> Import Project
  2. Select the folder containing the source for the QuizApp - UI theme
  3. Click Open and the project will build

Not the Project will build with the default google-services.json, you can change this to your own google-services.json when you need to modify the functionalities.

Create Firebase Project

  1. Follow here, sign in with your Goole Account



  2. Click add project link shown bove to open the new project conole.



  3. Enter your project name, select Country/region and accept terms of service and clic create project.




  4. Once yout project is created, you will see the your new proejct is ready, click continue to continue.




  5. Click on the option with Androi Icon (Add FIrebase to yur project) and the new project wizard will show up.




  6. Enter the App package name "me.jerryhanks.quizApp" or your own package name if you have changed it, App nickname and debug signing certificate SHA-1. Click on the ? icon beside the label to find out how to get the signing keys for debug and release on Mac osX and Windows. once you are done, click register app.




  7. If you are here, it means that your project is ready, download the googple-services.json and add it to the "app/" folder. Click next.




  8. You are done, once you have added the google-service.json, build your project.
